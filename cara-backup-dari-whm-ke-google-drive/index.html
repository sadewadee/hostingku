Cara membuat backup data dari cPanel ke Google Drive

Mem-backup file di cloud, sampai saat ini merupakan cara yang efisien untuk memastikan ketersediaan. Alternatif solusi lain pencadangan cloud adalah dengan membuat banyak salinan file atau biasa disebut sebagai redudensi dan menyimpan data tersebut di lokasi yang berbeda. Dengan cara ini, kamu tidak perlu khawatir file akan hilang jika mengalami serangan hacker, bad sector pada hard disk atau mengalami bencana alam. Dengan tutorial ini kamu bisa mempelajari cara untuk membuat cloud backup menggunakan google drive di whm.

Membuat Project baru dan API Google Drive

Untuk menggunakan fungsi cloud backup pada Google Drive kamu memerlukan sebuah project dan API Credentials. Semua ini dapat dilakukan dengan sangat mudah melalui Google Developers Console yang dapat kamu buka melalui akun googlemu. Untuk membuat proyek baru, klik tombol Proyek Saya / Pilih Proyek di bagian atas halaman.

Namun Jika ini adalah pertama kalinya membuka konsol, kamu tidak akan memiliki proyek aktif, jadi mari kita buat satu. Klik pada tombol Proyek Baru .

Beri nama proyek lalu klik tombol Buat

Sekarang mari kita aktifkan Google Drive API, jadi klik salah satu ikonnya jika ada atau tekan tombol Enable APIS and  Services .

Pilih opsi Enable untuk melanjutkan.

Karena nanti kita akan membutuhkan 2 kunci yang berbeda yang akan bertindak sebagai kunci publik dan pribadi kita, maka kita akan menambahkan kredensial untuk API, jadi klik tombol Create Credentials.

Di sini kamu akan melihat banyak opsi untuk setiap bagian. Yang kamu butuhkan adalah sebagai berikut:

  • API mana yang akan kita gunakan?
    API Google Drive
  • Dari mana kita akan memanggil API?
    Server web (mis. Node.js, Tomcat)
  • Data apa yang akan kita akses?
    Data pengguna

Setelah selesai, klik opsi ” What Credentials do I need? ” Untuk melanjutkan

Sebagai bagian dari tahap 2 pengaturan ini, kita akan diminta untuk membuat OAuth 2.0 Client ID.

Tambahkan nama yang mudah diingat dan deskriptif tentang penggunaan kredensial dan di URL pengalihan tambahkan hxxps: // hostname: 2087 / googledriveauth / callback dimana hostname akan diganti dengan hostname server. Perhatikan bahwa menggunakan IP server tidak diperbolehkan.

Klik pada Create OAuth Client ID, dan setelah itu, kita telah menyelesaikan bagian ke 3 dari pengaturan. Pada bagian 4 akan mendapatkan Client ID dan sisa kredensial untuk API. Pastikan untuk mengunduh file .JSONkarena akan diperlukannya untuk ” Private Key” atau client_secret. Klik Selesai untuk menyelesaikan pengaturan.

Buka file yang sudah di download dengan editor teks dan copy client _secret. Kita akan memerlukan ini untuk konfigurasi Google Drive di bagian kedua tutorial ini.

Tambahkan Google Drive sebagai Additional Destination

Sekarang kita akan melakukan pengaturan agar setiap backup yang dibuat oleh server dapat ditransfer ke Google Drive . Pertama kita ke WHM → Backup → Backup Configuration dan beralih ke tab Additional Destinations. Kemudian pilih Google Drive sebagai Destination Type.

Pada bagian Folder isi sesuai dimana kita menyimpan file backup yang dibuat oleh WHM biasanya file tersebut di tempatkan pada /backup. pada bagian Client ID diisi dengan Client ID yang kita dapatkan diatas. Hal yang sama berlaku untuk Secred Clinet yang bisad dodapatkan di file .JSON.

Setelah kita menyalin kedua kredensial API di bidang yang benar, klik tombol Generate Credentials. Ini agak seperti kebalikan dari apa yang baru saja kami lakukan di konsol Google Developers, dan sekarang Anda akan diminta untuk Authorize Access

Lanjutkan Membaca  Perbedaan VPS, Shared Hosting dan Dedicated server

Saya sarankan untuk membiarkan settingan pada kolom Timeout secara default selanjutnya klik tombol Save and Validate Destination. Jika hanya mengklik Save Destination , validasi dapat dilakukan kapan saja, tetapi validasi diperlukan agar file backup benar-benar sampai ke Google Drive.

Dengan ini, kita sudah menyelesaikan tutorial untuk melakukan backup dari WHM ke Google Drive, dengan menambahkan Google Drive sebagai cloud backup ke server kita maka akan meningkatkan juga level keamanan.
