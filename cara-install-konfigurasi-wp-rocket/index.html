Cara Install & Konfigurasi WP Rocket

Bagaimana ketika sedang membuka sebuah website dan website tersebut memerlukan waktu ratusan bahkan ribuan menit hanya untuk membuka satu halaman?, menyebalkan bukan?.

 

Sebuah penelitian dari Forrester Consulting mengatakan bahwa “47% orang mengharapkan halaman web dapat dibuka dalam dua detik atau kurang” .

Sayangnya banyak pemilik websiten tidak menyadari bahwa website yang lambat tidak hanya membuat orang frustasi namun dapat juga berdampak negatif pada peringkat Google dan tentunya akan sangat berdampak dengan koneversi.

Untungnya ada sebuah cara dimana kita bisa mempercepat loadtime website, terutama jika websitemu dibangun menggunakan wordpress. Disini saya akan membantumu bagaimana cara menggunakan WP Rocket.

Apa itu WP Rocket?

Sebelum kita mulai cara menggunakan WP Rocket mari kita berkenalan dulu dengan WP Rocket. WP Rocket adalah plugin caching premium untuk WordPress. WP Rocket sangat mudah untuk digunakan, dan cukup banyak settingan yang direkomendasikan dan perlu dikonfigurasi untuk mendapatkan hasil maksimal. Dibeberapa kasus bahkan bisa mengurangi loadtime hinggal 90%.

Plugin WP Rocket tidak seperti plugin SEO wordpress lainya yang memiliki fungsi yang sama. Seperti plugin W3 Total Cache dan WP Fastest Cache, mereka hadir dengan menawarkan versi gratis. Namun WP Rocket tidak demikian karena mereka yakin bahwa fitur plugin ini benar-benar berfungsi dengan baik untuk situs wordpress anda.

Namun sebelum itu, pastikan anda sudah membelinya di situs resmi WP Rocket untuk mendapatkan harga yang murah dan plugin yang asli.

Rekomendasi Settingan Terbaik WP Rocket

Untuk masuk ke halaman pengaturan WP Rocket kamu bisa ke Setting -> WP Rocket (Pengaturan -> WP Rocket), di halaman tersebut akan ada 10 tab yang harus di konfigurasi agar websitemu bisa cepat.

WP ROCKET DASHBOARD

Dashboard memberi informasi tentang lisensi dan kapan kedaluwarsanya. Kamu juga dapat ikut serta menjadi Rocket Tester (program pengujian beta) dan Rocket Analytics (memungkinkan WP Rocket untuk mengumpulkan data secara anonim).

Pada halaman dashboar kamu dapat menggunakan Remove All Cached Files (sangat disarankan saat sudah melakukan konfigurasi pada WP Rocketmu), Start Cache Preloading (untuk membuat cache pada halaman Home dan semua link yang ada dihalaman Home) dan Purge OPcache Content ( membersihkan OPcahce untuk mencegah terjadinya error saat mengupdate WP Rocket).

WP ROCKET CACHE SETTINGS

1. Enable caching for mobile devices harus diaktifkan karena memungkinkan caching untuk handphone dan membuat websitemu menjadi lebih cepat saat diakses. Fitur Ini wajib diaktifkan karena mengingat banyak penggunjung website yang berbasiskan mobile

Aktifkan juga Separate cache files for mobile devices. Karena caching seluler WP Rocket berfungsi paling aman dengan kedua opsi ini diaktifkan. Jika ragu, biarkan keduanya.

2. Enable caching for logged in WordPress users, aktifkan ini jika websitemu menggunakan sistem membership seperti woocommerce.

3. Cache lifespan secara otomatis diatur ke 10 jam yang berarti bahwa file yang di-cache dihapus secara otomatis setiap 10 jam sekali. Jika kamu menggunakan website landing page atau website yang jarang diperbaharuai, kamu bisa mengubahnya menjadi 1 hari sekali.

OPTIMASI FILE JS DAN CSS

Minifying file berfungsi untuk mengurangi ukuran file dan sehingga mengurangi waktu yang diperlukan oleh browser untuk memuatnya. Minifikasi menghapus spasi dan komentar dari file statis, sehingga memungkinkan browser dan search engine dapat memproses file HTML, CSS, dan JavaScript dengan lebih cepat.

Combining file akan menggabungkan file ke dalam 1 grup file yang lebih kecil untuk memastikan kompatibilitas tema / plugin dan kinerja yang lebih baik. Namun tidak disarankan untuk menggabungkan beberapa file menjadi 1 atau 2 file dengan ukuran besar, karena browser lebih cepat mendownload 6 file kecil secara paralel dari pada 1-2 file besar.

Menggabungkan CSS dan JS menjadi lebih sedikit file dianggap sebagai cara terbaik HTTP / 1, namu belum tentu demikian dengan HTTP / 2. Jika websitemu menggunakan HTTP / 2, berikut adalah hal-hal yang harus Anda pertimbangkan ketika mengkonfigurasi WP Rocket untuk HTTP / 2 .

1. Minify file HTML dengan mengaktifkan ini akan menghapus spasi dan komentar untuk mengurangi ukuran halaman websitemu.

2. Combine file Google Fonts akan mengurangi jumlah permintaan HTTP (terutama jika kamu menggunakan banyak font).

3. Remove query strings dari static files dapat meningkatkan kinerja di GT Metrix. Settingan ini akan menghapus query string pada file statis (misalnya style.css? Ver = 1.0) dan mengubahnya menjadi nama file  (misalnya style-1-0.css).

Lanjutkan Membaca  Website Ngebut Pakai LiteSpeed Cache

4. Minify file CSS dengan mengaktifkan ini akan menghapus spasi dan komentar untuk mengurangi ukuran file css.

5. Combine file CSS menggabungkan semua file Anda menjadi hanya satu file, yang akan mengurangi jumlah permintaan HTTP. Tidak disarankan jika situs Anda menggunakan HTTP / 2.

6. Optimize CSS delivery dengan menghilangkan render-blocking CSS di websitemu akan mempercepat loadtime. artinya websitemu akan load dalam keadaan tidak menggunakan css, Google PageSpeed Insights menilai hal ini.

Critical path CSS berarti halaman websitemu akan di-load tanpa CSS-ny, sehingga websitemu akan terlihat sedikit berantakan untuk beberapa saat sampai webistemu di-load sempurna oleh browser. Masalah ini biasanya disebut sebagai FOUC (flash of unstyled content) . Untuk menghindari hal ini, kamu harus mengaktifkan Critical Path CSS. Dengan meletakan CSS yang diperlukan oleh halaman itu dibagian atas, maka akan menghindari FOUC.

7. Minify file JavaScript menghapus spasi dan komentar untuk mengurangi ukuran file JS.

8.Combine JavaScript files Dengan menggabungkan file JavaScripts akan membuat lebih sedikit HTTP Request. Tidak disarankan jika situs Anda menggunakan HTTP / 2.

9. Load JavaScript deferred menghilangkan render-blocking JS diwebsitemu dan dapat meningkatkan waktu loading. Poin ini salah satu pertimbangan penilaian dari Google PageSpeed Insights.

10. Safe Mode for JQuery memastikan dukungan untuk referensi jQuery inline dari tema dan plugin dengan memuat jQuery terlebih dahulu sebagaim render-blocking script.

 

PENGATURAN MEDIA WP ROCKET

1. Lazyload Image berarti gambar hanya akan dimuat ketika keseluruhan halaman sudah dimuat atau saat gambar memasuki viewport. Lazyload berguna untuk mengurangi jumlah HTTP Request.

2. Lazyload iframe and video tidak berbeda jauh dengan poin satu, fungsi utamanya adalah mengurangi jumlah http request dan mempercepat loading.

3. Replace Youtube Iframe with preview image mengubah iframe youtube dengan gambar thumbnail sehingga mempercepat loading website.

Lazyload dapat dimatikan pada masing – masing halaman / posting (Anda menemukan pengaturan ini di sidebar posting / halaman)

4. Disable Emoji Menonaktifkan cache emoji mengurangi jumlah permintaan HTTP yang dapat meningkatkan waktu pemuatan.

6. Embed WordPress harus dinonaktifkan untuk mencegah orang lain mengambil konten dari website Anda.

PENGATURAN PRELOAD WP ROCKET

1. Preloading sitemap membuat preloading berdasarkan URL yang ada pada file sitemapXML.

2. Prefetch Permintaan DNS memungkinkan melakukan resolve DNS terjadi secara paralel dengan pengambilan konten halaman yang sebenarnya.

Anda dapat menentukan host eksternal (seperti //fonts.googleapis.com & //maxcdn.bootstrapcdn.com) yang akan di-prefetch karena prefetching DNS dapat membuat file eksternal dimuat lebih cepat, terutama pada jaringan seluler.

URL yang paling umum untuk prefetch adalah:

  • //maxcdn.bootstrapcdn.com
  • //platform.twitter.com
  • //s3.amazonaws.com
  • //ajax.googleapis.com
  • //cdnjs.cloudflare.com
  • //netdna.bootstrapcdn.com
  • //fonts.googleapis.com
  • //connect.facebook.net
  • //www.google-analytics.com
  • //www.googletagmanager.com
  • //maps.google.com

PENGATURAN ADVANCE RULE WP ROCKET

Pengaturan ini untuk manajemen cache tingkat lanjut, biasanya untuk mengecualikan halaman cart dan checkout di website e-commerce.

1. Never cache URL memungkinkan untuk menentukan URL halaman atau posting yang tidak boleh di-cache.

2. Never cache Cookies memungkinkan Anda menentukan ID cookie yang, ketika diatur di browser pengunjung, harus mencegah halaman dari di-cache.

3. Never Cache User Agents memungkinkan untuk menentukan User Agent yang seharusnya tidak pernah melihat halaman dalam cache.

4. Always Purge URL memungkinkan untuk menentukan URL yang selalu ingin Anda hapus dari cache setiap kali Anda memperbarui posting atau halaman apa pun.

5. Cache Query Strings memungkinkan Anda menentukan string kueri untuk caching.

 

PENGATURAN BASIS DATA WP ROCKET

Bagian ini dilengkapi dengan berbagai pengaturan untuk membersihkan dan mengoptimalkan WordPress.

1. Post Cleanup menghapus revisi, konsep otomatis, dan posting dan halaman yang dibuang. Hapus ini kecuali Anda memiliki versi lama dari posting (atau posting yang dihapus).

2. Comments Cleanup menghapus permanent spam dan komentar.

3. Transient Cleanup menghapus data option sementara yang tersimpan pada database.

4. Database Cleanup mengoptimalkan tabel dalam basis data WordPress.

5. Automatic Cleanup. Pembersihan dapat dilakukan berkala seminggu atau sebulan sekali, dengan adanya automatic cleanup, anda tidak perlu bersusah payah untuk melakukannya.

PENGATURAN CDN WP ROCKET

Menggunakan jaringan pengiriman konten (CDN) berarti bahwa semua URL file statis (CSS, JS, gambar) akan ditulis ulang ke CNAME yang Anda berikan.

Lanjutkan Membaca  Website Ngebut Pakai LiteSpeed Cache

1. Enable CDN . Aktifkan ini jika Anda menggunakan content delivery network. WP Rocket kompatibel dengan sebagian besar CDN seperti Amazon Cloudfront, MaxCDN, KeyCDN (yang saya gunakan) dan lainnya. Cari tahu lebih lanjut tentang cara menggunakan WP Rocket dengan CDN

2. CDN CNAME . Salin CNAME (domain) yang diberikan kepada Anda oleh penyedia CDN Anda dan masukkan ke dalam CDN CNAME. opsi ini akan meng-over write semua URL file statis.

3. Exclude file tidak semua file bisa menggunakan CDN, misalnya file video yang memiliki ukuran besar, Dengan opsi ini anda bisa melakukan pengecualian..

 

ADD-ON WP ROCKET (CLOUDFLARE)

WP Rocket memungkinkan Anda mengintegrasikan akun Cloudflare Anda dengan fungsi tambahan mereka.

1. Global Api Key . Anda akan menemukan kunci API di kanan atas di akun Cloudflare Anda. Cukup buka profil Anda dan gulir ke bawah dan Anda akan melihat kunci API global Anda. Anda hanya perlu menyalin dan menempel ini ke WP Rocket.

2. Account Email . Ini adalah alamat email yang Anda gunakan untuk login ke Cloudflare.

3. Domain . Ini adalah nama domain Anda, mis. sadewa.my.id.

4. Developer Mode . Aktifkan sementara mode pengembangan di situs web Anda. Pengaturan ini akan mati secara otomatis setelah 3 jam. Mode ini sangat bagus ketika anda sedang melakukan banyak perubahan tampilan pada website

5. Optimal Setting . Secara otomatis meningkatkan konfigurasi Cloudflare Anda untuk kecepatan, tingkat kinerja, dan kompatibilitas. Opsi ini mengaktifkan pengaturan Cloudflare yang optimal.

6. Relative Protocol . Hanya digunakan ketika fitur SSL Cloudflare yang fleksibel. URL file statis (CSS, JS, gambar) akan ditulis ulang untuk menggunakan // daripada http: // atau https: //.

WP ROCKET TOOLS

1. Pengaturan ekspor memungkinkan Anda mengekspor pengaturan WP Rocket Anda untuk digunakan di situs lain.

2. Pengaturan Impor memungkinkan Anda mengimpor pengaturan WP Rocket Anda yang telah dikonfigurasi sebelumnya.

3. Rollback memungkinkan Anda kembali ke versi sebelumnya, jika versi baru WP Rocket menyebabkan masalah bagi Anda.

 

HOST WEB MANA YANG BERFUNGSI DENGAN WP ROCKET?

WP Rocket kompatibel dengan hampir semua host web . Namun beberapa, terutama host WordPress yang dikelola, mungkin tidak berfungsi dengan WP Rocket. Jika penyedia hosting Anda tidak tercantum di sini di bawah ini, ini tidak berarti itu tidak kompatibel dengan WP Rocket. Cara terbaik untuk memastikan 100% adalah dengan menghubungi host web Anda dan bertanya.

  • Kinsta : Kinsta hanya mendukung WP Rocket versi 3.0 dan lebih tinggi. Caching halaman WP Rocket secara otomatis dinonaktifkan untuk mencegah konflik dengan caching bawaan Kinsta. Kinsta adalah mitra resmi WP Rocket.
  • WP Engine : WP Rocket adalah satu-satunya plugin caching yang diizinkan di WP Engine. WP Engine adalah mitra resmi WP Rocket.
  • SiteGround : WP Rocket kompatibel dengan caching statis, dinamis, dan mem-cache SiteGround. SiteGround adalah mitra resmi WP Rocket.
  • A2 Hosting : WP Rocket sepenuhnya kompatibel dengan A2 Hosting . Tetapi Anda harus menginstal WordPress di situs Anda sebelum dapat menginstal plugin WP Rocket. A2 Hosting adalah mitra resmi WP Rocket.
  • WebHostFace : WebHostFace mendukung (dan merupakan mitra resmi) WP Rocket.
  • Savvii : Savvii mendukung (dan merupakan mitra resmi) WP Rocket.
  • FastComet : Menawarkan paket yang dioptimalkan secara khusus untuk WordPress dan WP Rocket. FastComet adalah mitra resmi WP Rocket.
  • Paket Bluehost Managed WordPress : Bluehost Managed WordPress berencana konfigurasi Varnish merusak minifikasi WP Rocket, jadi Anda harus mematikan Bluehost’s Varnish , atau mematikan minification WP Rocket.
  • Cloudways WordPress Hosting : Saat menggunakan minifikasi WP Rocket dengan Varnish Cloudways, Anda harus membuat aturan pengecualian untuk Varnish dalam pengaturan aplikasi Cloudways.
  • Roda gila : Anda harus menghubungi dukungan Roda Gila dan meminta mereka untuk mengaktifkan WP Rocket.
  • Paket WordPress yang Dikelola HostGator : WP Rocket tidak diizinkan di hosting WordPress yang Dikelola oleh HostGator .
  • Sintesis : W3 Total Cache telah diinstal sebelumnya pada Sintesis tetapi dapat dihapus dan diganti dengan WP Rocket.
  • WebSavers.ca : WebSavers.ca adalah mitra resmi WP Rocket.

Baca lebih lanjut tentang host web yang kompatibel dengan WP Rocket di https://docs.wp-rocket.me/article/670-hosting-compatibility .
