Celah Keamana Plugin Themegrill Demo Importer

Pertengahan Februari 2020 ada kabar yang tidak mengenakan datang dari plugin ThemeGrill Demo Importer. Plugin millik ThemeGrill ini memiliki fungsi untuk melakukan import demo content dengan sekali klik, sehingga memudahkan pengguna untuk mengimpor konten demo, widget, dan pengaturan dari ThemeGrill, membuat pengguna dapat menyesuaikan tema dengan cepat dan mudah.

Menurut data wordpress, plugin ini sudah diinstall dilebih dari 200.000 website aktif dan parahnya belum semua terupdate pada versi terbaru. Plugin ini memiliki celah yang bisa dieksploitasi oleh peretas dari jarak jauh untuk menghapus website dan mendapatkan hak akses sebagai administrator pada website. Dari hasil investigasi WebARX celah ini ada pada plugin versi 1.3.4 hingga versi sebelum 1.6.1.

Dari laporan WebARX ketika menginstall dan menjalankan plugin ThemeGrill Demo Importer, plugin akan mengeksekusi sebuah fungsi tanpa melakukan pengecekan autentikasi sebagai admininstrator. Bug ini menyebabkan peretas dapat dengan mudah masuk sebagai administrator tanpa perlu melakukan autentikasi. Serangan ini menggunakan payload yang ditargetkan pada website yang menggunakan theme Themegrill dan username admin sebagai administratornya.

Dari script diatas kita dapat melihat bahwa tidak ada pemeriksaan autentikasi, dan hanya ada parameter do_reset_wordpress yang perlu dijalankan pada halaman admin.

Menurut peneliti keamana WebARX celah ini adalah celah keamanan yang sangat berbahaya serta menimbulkan banyak kerusakan. Karena tidak memerlukan payload yang mencurigakan untuk melakukan eksekusi, firewall tidak dapat diharapkan untuk memblokir secara default dan perlu pengaturan khusus untuk memblokir dan mencegah masalah ini dikemudian hari.

Pencegahan dan Cara Mengatasi Masalah Keamana Plugin Themegrill Demo Importer.

Untuk anda yang belum terkena dampak dari celah tersebut berikut beberapa tindakan antisipasi agar tidak terjadi pada website anda :

1. Selalu lakukan backup

Selalu gunakan fasilitas backup yang disediakan oleh penyedia hosting untuk memastikan website tetap aman dan terjaga serta mecegah hal –  hal buruk terjadi. Jika memungkinkan sewa cloud backup dengan layanan backup harian untuk memastikan website lebih aman

2. Ubah Username dan Password

Pada kasus bug plugin ThemeGrill Demo Importer, yang menjadi korban adalah website yang menggunakan username “admin”, gunakan username yang tidak menggunakan kata yang umum dan sama dengan nama website atau domain anda. Selain itu dalam pengunaan password, gunakan kombinasi dengan angka dan huruf dan spesial karakter agar lebih kuat.

3. Lakukan Update Rutin

Tujuan dari diadakannya Update adalah untuk mengatasi permasalahan dan bug pada versi sebelumnya. Tidak hanya plugin, theme dan wordpress juga perlu untuk di update secara rutin agar terhindar dari masalah. Ketika plugin ThemeGrill Demo Importer sudah merilis versi 1.6.2, celah keamanan yang sebelumnya juga sudah ditutup sehingga anda aman dan tidak perlu khawatir menjadi target serangan.

Jika anda sudah menjadi korban dari bug Plugin Themegrill Demo Importer salah satu cara untuk mengembalikan website seperti semula adalah meminta penyedia hosting untuk melakukan restore database website anda.

Perlu digaris bawahi bahwa setiap sistem pasti memiliki celah keamanan yang bisa dieksploitasi oleh para peretas, tetapi Anda bisa mencegahnya dengan mengikuti 3 langkah diatas. Demikian pembahasan dari saya, terima kasih.
