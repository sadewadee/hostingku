Menghitung Trafik Pada Hosting

Saat kita sedang menjalankan kampanye atau hal lain yang dapat menghasilkan trafik yang jauh lebih tinggi daripada biasanya, kita harus mempersiapkan dan memperkirakan dampaknya pada hosting. Ada banyak variabel yang dapat memengaruhi kinerja website serta meningkatkan beban pada server – dan dalam artikel ini saya akan menguraikan apa yang harus di khawatirkan, apa yang tidak perlu dikhawatirkan, serta bagaimana membuat perkiraan dampak dari kampanyemu tentang penggunaan sumber daya server.

Selalu ada timbal balik antara kinerja dan skalabilitas. Ketika membuat website dengan lalu lintas tinggi, kita harus memprioritaskan skalabilitas sebelum kinerja pada end user, kita sering kali tidak mendapatkan keduanya. Hosting terdiri dari banyak layanan yang bekerja bersama untuk menyediakan halaman web kepada end user, dan semua hosting memiliki batasan.

Rumus ajaib dari Google Analytic yang saya gunakan untuk memberi perkiraan jumlah pengguna yang dapat ditangani oleh website saya adalah:

(jumlah core CPU / Rata-rata waktu respon halaman dalam detik) * 60 * Frekuensi Klik Pengguna (detik) = Jumlah Pengujung secara bersamaan

Performa halaman depan dapat kita ukur dengan menggunakan Pingdom Tools atau PageSpeed Insights namun hal tersebut tidak terlalu menjadi masalah. Masalah sebenarnya bukan bagaimana halaman depan kita bekerja, tapi seberapa banyak traffic dan beban yang dapat ditangani oleh hosting anda.

Cara memperkirakan kapasitas Hosting

Saya punya beberapa metode sederhana untuk menghitung perkiraan berapa banyak traffic yang dapat di tangani oleh hostingmu. Permasalahan yang pertama kali kita hadapi saat mengingkatkan traffic adalah PHP dan Database. Ada banyak cara untuk mengurangi beban pada dua hal ini, tapi mari kita perhitungkan dulu.

Cara memeriksa jumlah core CPU pada hosting.

Saat kita memesan hosting, kita bisa menanyakan berapakah jumlah core CPU yang akan kita dapatkan, karena semakin banyak jumlah core akan semakin baik untuk website kita. Mari kita gunakan Intel E5 sebagai contoh.

bash-4.2 $ prosesor grep / proc / cpuinfo | wc -l
16

Berapa lama rata-rata waktu CPU yang diperlukan saat melakukan request PHP?

Data lain yang kita butuhkan untuk membuat perkiraan adalah jumlah rata-rata waktu yang digunakan CPU untuk merender atau memunculkan halaman pada website Anda. Cara paling sederhana untuk melakukan ini adalah dengan memeriksa beberapa halaman yang berbeda lalu  hitung rata-rata.

Angka yang saya gunakan untuk estimasi adalah jumlah Time To First Byte (TTFB) + Content Download Time. Pada contoh di bawah ini kita dapat melihat bahwa waktu yang diperlukan untuk memuat halaman depan adalah 526 ms yaitu 0,5 detik.

Untuk toko E-commerce, penting untuk memeriksa performa halaman depan, halaman kategori, halaman produk, keranjang belanja dan halaman checkout serta membuat perkiraan. Jika website menggunakan caching diseluruh halaman, Kita harus menambahkan cache buster (cukup tambahkan beberapa parameter ke url, misalnya https://sadewa.my.id/?cache=busted). Ini akan membuat request tanpa mengunakan cache.

Apa hubungan antara CPU Core dan Request time?

Ketika pengunjung mengklik halaman website, server hosting akan bekerja dengan merender halaman website sampai kita mendapatkan tampilan yang sesuai. Misalkan dengan contoh di atas, CPU akan sibuk selama 526 milidetik untuk menghasilkan halaman depan website ini.

Formula untuk menghitung kapasitas maksimal server web Anda

Jumlah inti CPU / Waktu rata-rata untuk permintaan halaman (dalam detik) = Jumlah Maks. Permintaan Halaman per detik

Kapasitas server hosting adalah 16 core CPU, jadi ketika setiap permintaan ke situs web rata-rata menggunakan waktu CPU 0,526 – kita mungkin mengharapkannya untuk dapat menangani sekitar 16 core / 0,526 detik Waktu CPU = 30 permintaan per detik.

Mengapa permintaan halaman per detik merupakan metrik penting untuk Skalabilitas?

Skalabilitas situs web kita biasanya berakhir saat server hosting kita mencapai batas CPU. Average time request didasarkan pada waktu yang diperlukan untuk melakukan query pada database dan respon php dalam mengirimkan data.

Berapa banyak pengunjung yang dapat berkunjung ke website saya secara bersamaan?

Pertanyaannya adalah apa kaitannya antara page views per second dengan Google Analytics? Untuk mengetahui berapa banyak pengunjung yang bisa datang secara bersamaan pada websitemu, kita perlu memeriksa Google Analytics dan menghitung seberapa sering pengunjung mengklik.

Jika kita menuju ke Audiens> Ikhtisar, maka akan menemukan Average Session Duration dan Pages per Session.

Session Duration dalam contoh ini adalah 9 menit 10 detik atau sama dengan 9 × 60 + 10 = 550 detik. Setiap pengguna rata-rata mengklik 4,82 kali. Bagilah Average Session Duration / Page per Session, dan kita akan mendapatkan 114 detik. Ini berarti bahwa rata-rata pengunjung website kita mengklik kira-kira sekali setiap 2 menit (setiap 114 detik). Jumlah ini bisa sangat bervariasi, tetapi untuk E-commerce aturan praktisnya adalah sekali per menit.

Ketika saya tahu bahwa pengunjung di Google Analytics mengklik sekali setiap detik, dan saya tahu bahwa kapasitas server adalah 30 permintaan per detik, saya dapat menghitung perkiraan yang cukup tepat;

Jumlah permintaan maks per detik * 60 * Frekuensi klik per detik = Jumlah Maks Pengunjung Bersamaan

30 permintaan per detik * 60 detik * Frekuensi klik dalam beberapa menit 2 = 3600 pengunjung bersamaan di Google Analytics

Ada banyak pertanyaan yang dapat kalian ajukan mengenai cara penghitungan ini, tetapi dari pengalaman saya cara menghitung ini memberikan perkiraan yang cukup tepat.

Berapa banyak pengunjung dengan VPS saya saat ini?

Jika kita menggunakan VPS untuk website dan hanya memiliki satu atau beberapa core CPU, kita bisa menggunakan hitungan sebagai berikut. Sebagai contoh:

Average PHP request time : 550 ms
Core CPU : 2
Frekuensi klik: 45 detik (perkiraan)

2 core / 0,65 = 3.6 tampilan halaman per detik * 60 * 0,75 = 327 Pengunjung bersamaan

Cara meningkatkan skalabilitas situs web Anda

Pada dasarnya ada dua hal yang dapat kita lakukan untuk meningkatkan skalabilitas website. mengurangi konsumsi sumber daya per pengunjung, atau meningkatkan sumber daya server kita.

Untuk sumber daya pada hosting server untuk E-commerce sangatlah penting, karena sebagian besar permintaan bersifat dinamis dan akan menjalankan PHP. Caching akan sedikit membantu, tetapi dengan banyak session dan cart, sumber daya server biasanya akan dihabiskan dengan cepat pula. Oleh karena itu, ide terbaik yang bisa kita gunakan adalah membuat sebuah website ecoomerce dengan sedikit menggunakan sumber daya dan buat website menjadi lebih cepat.

Saat sebuah website menggunakan sumber daya melebihi batas, tidak hanya akan selesai jika melakukan upgrade ke paket hosting atau server yang lebih tinggi namun akan menambah biaya pada konsultasi, pengembangan dan banyak hal rumit lainnya.
